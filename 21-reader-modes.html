01

Complex logic that couldn't be simplified

SaaS·Segment Builder·Condition Logic

Marketing teams build audience segments out of layered attributes, events, and consent rules. I turned that tangle into a workspace they could read and trust.

The hard constraint: boolean logic had to stay intact. The challenge: non-technical users needed to own and audit their own segments without engineering help. I designed the structure that made the logic visible without pretending it was simpler than it was.

02

A migration with a system underneath

Mobile·Web·Migration UX

3BB Member was live for over a million users. My job was to ship only the flows where the migration changed the user's path.

Started by auditing every screen in a 200+ screen Figma file before designing anything. Identified 7 migration-affected flows. Designed those. Left the rest untouched — not because they were good, but because changing them would have added risk without migration benefit.

03

A redesign that had to feel like no change

Retail·Physical Terminal·Operational UX

POS terminals run on speed and muscle memory. Better hierarchy without moving what cashiers' hands already knew.

Position constraints were absolute. All improvement had to come from within the existing spatial footprint — through weight, contrast, and proximity grouping alone. The constraint wasn't a limitation. It was the design problem.
