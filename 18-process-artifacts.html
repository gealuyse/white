01 Customer Data Platform — Segment Builder Case study →

"marketers need to own their own segments. NO engineering dependency."

stakeholder feedback · week 1
Original brief excerpt

Users need to build audience segments using boolean logic (AND/OR/NOT) with support for attributes, events, and consent flags. The system must support nested condition groups and preview segment size before saving.

49 attributes. most users knew maybe 12 of them.

Final segment builder — readable boolean tree

Should the condition builder be free-form text or a guided structured form?

Went with guided structure — tested free-form with 3 non-technical users who couldn't verify if the system had understood their logic. The builder made the result readable at a glance. Power users got field-level operators. Everyone else got legibility.

"how does a user know if AND applies to the whole group or just the row above it?"

02 3BB Member App & Website Case study →
Handoff situation

Received a Figma file with 200+ screens. Multiple pages, unclear which was current. App live in production, migration scheduled in 6 weeks.

audited every screen before touching anything

"scope to migration-affected flows ONLY. anything else is risk with no benefit."

PM alignment · day 3

3BB Member — redesigned migration flows

Full redesign of account management, or surgical changes to migration-touched flows only?

Surgical changes only. Full redesign would have touched 60+ screens and added 3 months. The migration impacted 7 key flows. Designed those. Left the rest — not perfect, but the right risk trade for a live product.

03 Counter Service POS Terminal Proposal →

"do NOT move anything. cashiers are trained on this exact layout."

ops manager · kickoff
Design constraint

Every element must stay in its existing position. No new features. No new flows. Improvement must come from within the existing spatial footprint.

this IS the design challenge

If layout can't change, what CAN we improve?

Typography weight, contrast, and proximity grouping. Lighter inactive states. Stronger active states. Visual hierarchy without spatial change. The same screen — but readable at speed.
