# PART 2 — Margin Voice Spec (current state)

## What it is
One annotation voice (the old two-layer blue/red system is deprecated). Hand-written margin notes (Caveat), **single ink tone**, that either show the reasoning behind a decision or make a hard word plain. Not redlines, not decoration — the closest thing on the page to you thinking out loud, edited to one line.

## The four principles
1. **Anchor at the work.** A pointer to a real spot, never a floating block. *Test: if the note reads complete without seeing the work, it's a paragraph, not a margin note.*
2. **One sentence. No exceptions.** If it needs two, it's body copy.
3. **Replace, don't add.** Every note cuts the paragraph it replaces. *Test: you must be able to point at the paragraph it kills.*
4. **Why, not what.** Reasoning, not component names. *Test: if the note still reads true on another component, it's a label, not a reason.*

*Footnote to Rule 4 — soften at the user:* when the note's subject is the **person using the product** (not your own decision), drop the edge. Warmth, not wit. The one place the voice doesn't bite — and keeping it narrow is what makes the bite elsewhere credible.

*Meta-principle (the big one):* **the best annotation is sometimes rewriting the sentence so no annotation is needed.** Treat the cause, not the symptom. This is exactly why the landing lost almost all its marks — once the copy was plain, there was nothing left to gloss.

## Two classes — the rules apply differently
| Rule | Reasoning note | Clarity gloss |
|---|---|---|
| 1 Anchor | underline + solid line | circle + dashed line |
| 2 One sentence | yes | yes (often a fragment) |
| 3 Replace | **yes** — cuts a paragraph | **no** — purely additive, nothing to cut |
| 4 Why/what | why | **exception** — "what" allowed (defines a word for the reader) |

## Mark vocabulary — two independent axes
- **Axis A — the mark on the word (what kind of note):** underline = reasoning (phrase) · circle = gloss (single word)
- **Axis B — the line style (how heavy):** solid = primary, load-bearing · dashed = secondary, an aside

Default pairings: reasoning → solid, gloss → dashed. Mixable, so it's a real 2×2 grammar, not two fixed combos.

## The marks
| Mark | Glyph | Job | Status |
|---|---|---|---|
| Reasoning | underline + solid | why this choice was made | locked |
| Gloss | circle + dashed | a hard word, made plain | locked |
| Cut | strikethrough | what I removed, and why | locked |
| Gap | caret `^` | what's missing / what I'd add | locked |
| Cause | connector arrow | this forced that | locked · max 1 / page |
| Weight | double-underline **or** bold single arrow | the sharpest line on the page | locked · max 1 / page · **execution pending** |
| Group | brace `{` | this region is one unit | **pending** — glyph or fold into connector |

## Modes (not glyphs)
- **Illustration / doodle:** when the *system* is the story (not a screen), draw it hand-loose and annotate the parts — same four rules at object scale. One per case, max. More than that and it becomes noise.
- **Thesis note (page-level framing line):** **pending** — include one framing line per page, or not.

## Register rules (from the reference sweep)
- **Polish is voice.** Loose hand = "thinking" (Caveat is correct). Clean boxes + sharp leader lines = redline / spec sheet — avoid.
- **One ink tone, not many.** Multiple colors + competing systems = the over-annotated-article failure mode.
- **Density cap.** The moment everything is marked, nothing reads. The landing carries the fewest marks; the dissection marks (cut / cause / gap) live on case-study pages where real decisions exist.
- **The WHY test still filters even pretty annotations.** "USB hub" = naming (what) ✗. "rubber prevents leaking" = reasoning (why) ✓.

## Deliberately rejected
Checkmark / status ticks · asterisk-footnotes · standalone `?` as a glyph · numbered circles · highlight / marker fill.

## Budget per page

## The single landing demo mark (LOCKED)
- **Type:** reasoning mark (underline + solid line + note).
- **Anchor:** underline the phrase **"stops being the user's problem"** in the What-gets-easier line.
- **Note:** *do it well and no one sees it happened*
- **Why this spot:** it's theme #1 of the whole portfolio (complexity moves from the user to me); it's a quiet body line, not the already-loud headline; it passes the WHY test (says why good work is invisible, names nothing).
- **Upgrade path:** this spot was reserved for the *weight* mark. Weight's execution is still open, so it ships now as a single-line reasoning mark; once weight is decided it can be upgraded single → double underline on the same anchor.

## Reasoning-mark build spec (the component)
Three parts, reusable as one component so case-study pages can drop it in.

1. **Underline (anchor)** — wrap the phrase in `<span class="anno-anchor">`. The underline is a **hand-drawn stroke** (inline SVG or a squiggle background), single ink tone. **Never `text-decoration: underline`** — that reads as a link. Underline a *phrase*, not a whole sentence; one anchor per note.
2. **Connector (line)** — a hand-drawn, slightly curved line, single tone, from the anchor out to the note; small arrowhead optional. **Solid = reasoning.** Responsive: on desktop it reaches into the right margin where the note sits; on narrow/mobile the margin collapses, the line shortens, and the note drops *directly under* the anchor. **The line must never cross body text.**
3. **Note (text)** — Caveat, single tone, **one sentence ~8–12 words**. States the *why*, names no component (Rule 4). It **replaces** the paragraph that said the same thing (Rule 3). Real text, not an image (screen-reader accessible).

**Build rules:** single ink tone only (the deprecated blue/red two-layer is gone). Build it as one reusable component (anchor + line + note). **This pass implements the reasoning mark only** — gloss / cut / gap / cause come later; weight / brace / thesis are still undecided, don't touch.
