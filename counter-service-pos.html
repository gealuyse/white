Operational UX / POS

Counter Service POS

Everything was locked except spacing and contrast — and cashiers run on muscle memory, so the screen had to read better without making a single hand relearn where anything was.

Outcome
Review-approved spec
Timeline
2023
Role
UX/UI Designer

Cashiers process hundreds of transactions per shift. They don't read every screen from scratch — they rely on positions and muscle memory built over time.

The interface needed clearer hierarchy and more consistent spacing. But the flow was fixed. The button positions were fixed. The brand was fixed. The screens ranged from 800×600 to 1920×1080. Everything was locked except spacing, hierarchy, and contrast.

01

Improve hierarchy without moving the controls

I kept every button in place and adjusted hierarchy through spacing, grouping, contrast, and clearer separation between transaction areas and action areas.

— better, without making anyone relearn

02

Build layout rules for two screen realities

Separate layout profiles for compact 800×600 and expanded 1920×1080 displays — the compact version prioritized essential metadata; the wider version gave more room without simply enlarging everything.

— two profiles, one constraint made visible

"This project needs proof. The entire design rests on one claim."
