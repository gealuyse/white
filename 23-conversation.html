Work

Three situations where the system had to stay complex but the interface had to become clear.

01

Complex logic that couldn't be simplified

"Can you build segments that work on AND/OR/NOT logic, with nested groups, and show me how big the audience is before I save?"

I turned that into a structured builder where the logic stayed intact but became readable. Non-technical users could own their own configurations without engineering help.

02

A migration with a system underneath

"The app is live. We're migrating to a new backend. We need the affected flows redesigned — but nothing else should change."

Working inside a large inherited file. Identifying what was current. Shipping only the migration-affected flows. Leaving the rest exactly as it was.

Read case study →
03

A redesign that had to feel like no change

"Cashiers have been using this for years. Their hands know where everything is. Make it clearer — but don't move anything."

Every improvement had to come from within the existing spatial footprint. Weight, contrast, and proximity — no repositioning, no retraining required.
